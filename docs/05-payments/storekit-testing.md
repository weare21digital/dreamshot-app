# StoreKit Testing Guide

How to test in-app purchases locally using Xcode's StoreKit Configuration File.

## Option 1: StoreKit Configuration File (Simulator + Device)

This lets you test IAP without App Store Connect, sandbox accounts, or network.

### Create the file

1. Open Xcode workspace: `mobile-app/ios/mobileapp.xcworkspace`
2. File → New → File → StoreKit Configuration File
3. Name it `StoreKitConfig.storekit`
4. Save in `mobile-app/ios/` (don't commit — regenerated by prebuild)

### Add products

For each product in `src/config/iap.ts`, add a matching product:

1. Click **+** in the StoreKit file
2. Choose product type (Non-Consumable, Auto-Renewable Subscription, etc.)
3. Set **Product ID** to match exactly (e.g., `app_lifetime`)
4. Set a **Reference Name** and **Price**

### Enable in scheme

1. Product → Scheme → Edit Scheme
2. Run → Options → StoreKit Configuration → select your `.storekit` file
3. Build and run

Purchases will now work in simulator with instant confirmation (no Apple ID needed).

### Useful testing features

- **Transaction Manager** (Xcode → Debug → StoreKit → Manage Transactions): view/delete purchases
- **Clear Purchase History**: lets you re-test non-consumable purchases
- **Fail Transactions**: test error handling
- **Interrupted Purchases**: simulate Ask to Buy

## Option 2: Sandbox Testing (Device Only)

For testing against real App Store infrastructure.

### Prerequisites

- App binary uploaded to App Store Connect (TestFlight)
- IAP products created with status "Ready to Submit"
- Paid Apps Agreement active

### Create sandbox tester

1. App Store Connect → Users and Access → Sandbox Testers
2. Create account (use a `+` alias: `you+sandbox@gmail.com`)
3. On device: Settings → App Store → Sandbox Account → sign in

### Test flow

1. Build and install: `npx expo run:ios --device`
2. Navigate to premium screen
3. Purchase will use sandbox (no real charges)
4. Sandbox purchases auto-renew faster (subscriptions renew every few minutes)

### Sandbox subscription renewal rates

| Real Duration | Sandbox Duration |
|--------------|-----------------|
| 1 week | 3 minutes |
| 1 month | 5 minutes |
| 1 year | 1 hour |

## Device-mode notes

When `paymentMode: 'device'` (in `src/config/iap.ts`):
- StoreKit 2 handles verification locally — no backend needed
- Premium status saved to AsyncStorage (`@iap_device_premium_status`)
- Works in both simulator (with StoreKit Config) and device (with sandbox)
- Debug reset: set `@iap_debug_force_free` in AsyncStorage to reset premium

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Products return empty array | StoreKit Config not selected in scheme, or product IDs don't match |
| "Cannot connect to iTunes Store" | Using simulator without StoreKit Config — add one |
| Purchase completes but premium not detected | Check `useDevicePremiumStatus` hook and AsyncStorage key |
| Sandbox purchases not working | Re-sign into sandbox account, force quit app |
